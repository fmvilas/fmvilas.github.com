The importance of Open Source Web Applications

Online Web apps. It's a topic worrying me a long time ago. What we have? Where we go? Why? What should do we do? Web apps are here to stay, and to stay for a long time.

Back in the days

We have to look backward to understand what is happening here. We used to think in the Web as a big big collection of websites or, technically speaking, using servers as Hosting-as-a-Service (i know it's not the term). But what if we use servers as a Infrastructure-as-a-Service (IaaS)? Then we can extend our possibilities far far away. Nowadays we have an operating system, software, resources and more on the server side, and better browsers on the client side. We just need our imagination and a lot of work.

Big companies know it

Web apps are something that really amazes me, but I have to recognize that they can be an easy way to collect people information that sometimes can be used in bad ways. Big companies know it, and also know they can control software piracy through them. Ethical debates aside, we all use pirate software and one thing we should know it's that Web apps are imposible to pirate for a long time. I always use the same example: imagine you open your browser and browse for photoshop.com, then you enter your account name and password, and start using Photoshop right in your browser, storing your files in the cloud or in your computer.

Well, it's not only future, it's already here. To mention some examples:

  • Gmail/Hotmail. Email applications are probably the first Web apps.
  • Google Docs (now Drive). It can be a future replacement of Microsoft Office and even more.
  • Adobe Brackets. It can be a future replacement of NetBeans, Eclipse, Sublime Text, ...
  • Grooveshark. Your music everywhere and for free. And yes, they earn a lot of money.

And it's not only about software. For example Google created the Chromebook with their own operating system which is based on the Chrome browser and where all apps are Web apps.

What can we do?

It's at this point where open source software becomes relevant in the probably dirty area of the online Web apps. We need to look at the software as a public service and earn money in another way. I think we, as developers, should create or collaborate with open source projects. Now even more.

It's because of this I started Didgeridoo IDE, an open source project of a web-based IDE. I know that another alternatives exists, such as Cloud9 IDE but while it's nice it's not completely open source.

If we are moving to 'browser as an operating system' and we want open source Web apps, we should have an IDE working on the browser, being it offline or online.

Think forward, collaborate with open source software.
